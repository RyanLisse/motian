## Kandidaten

| Methode | Pad | Beschrijving |
|---------|-----|-------------|
| `GET` | `/api/candidates` | Lijst met zoeken (query, location, skills, role) |
| `GET` | `/api/candidates/:id` | Kandidaat ophalen |
| `PUT` | `/api/candidates/:id` | Kandidaat bijwerken |
| `DELETE` | `/api/candidates/:id` | Soft-delete kandidaat |
| `GET` | `/api/candidates/:id/matches` | Matches voor kandidaat |

## Validatie

Alle request bodies worden gevalideerd met Zod schema's uit `src/schemas/`. Bij ongeldige input wordt een `400` response teruggegeven:

```json
{
  "error": "Validation failed",
  "details": [
    { "path": ["email"], "message": "Invalid email format" }
  ]
}
```

## Foutcodes

| Code | Betekenis |
|------|-----------|
| `200` | Succes |
| `400` | Validatiefout (Zod) |
| `404` | Resource niet gevonden |
| `500` | Interne serverfout |
