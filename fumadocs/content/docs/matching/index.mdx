## Score Dimensies

### Rule-based scoring (Fase 1)

| Dimensie | Gewicht | Beschrijving |
|----------|---------|-------------|
| Vaardigheden | 40% | Matching kandidaat skills vs vereisten |
| Locatie | 20% | Afstand / provincie proximity |
| Tarief | 20% | Kandidaat uurtarief past binnen bandbreedte |
| Rol | 20% | Fuzzy match op functietitel |

### Hybrid Score

Wanneer embeddings beschikbaar zijn:

```
finalScore = (0.6 × ruleScore) + (0.4 × vectorScore)
```

Waarbij `vectorScore = cosineSimilarity(job, candidate) × 100`.

## AI Modellen

| Model | Rol | Fase |
|-------|-----|------|
| OpenAI text-embedding-3-small | Vector embeddings (512 dim) | 1 |
| Gemini Flash | Vereisten extractie + scoring | 2 & 3 |
| Grok (xAI) | Onafhankelijke judge verificatie | 4 |

## API

```bash
# Auto-match kandidaat met top 3 vacatures
POST /api/matches/auto
{ "candidateId": "uuid" }

# Auto-match vacature met top 3 kandidaten
POST /api/matches/auto
{ "jobId": "uuid" }

# Gestructureerde match uitvoeren
POST /api/matches/structured
```

Zie [API Referentie](/docs/api) voor volledige documentatie.
